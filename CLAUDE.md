# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A macOS menu-bar app called **Timap** (Time + Map) for cross-timezone meeting scheduling. The user is in an international company and needs to quickly see who's in working hours and find meeting windows.

The repo holds two layers:

- `prototype/` — a React + Babel HTML/JSX prototype generated by claude.ai/design. **Frozen design spec, not production code.** Use it as the visual source of truth for layout, colors, sizing, and copy. Do not modify it to fix product bugs — fix in the Swift app instead.
- `Timap/` — the actual macOS app: Swift Package Manager package using SwiftUI's `MenuBarExtra` scene (macOS 13+). This is what gets shipped.

The prototype was built first as a single big visual sketch with multiple direction variants (`direction-b/d/bd.jsx`, `bd-map-styles.jsx` style switcher). The user landed on the **B+D combined view with `dotGrid` map style** — that's what the Swift app implements.

## Common commands

Run from `Timap/`.

```sh
make            # build & wrap into dist/Timap.app
make run        # build, wrap, and `open` the app (menu-bar icon appears)
make verify     # run TimapCore correctness checks (substitute for unit tests)
make install    # copy dist/Timap.app to /Applications
make clean      # remove .build and dist
swift build     # debug build (no .app wrapping)
swift run TimapVerify   # equivalent to `make verify`
```

**There is no `xcodebuild` step and no `.xcodeproj`.** This codebase intentionally avoids the Xcode project file. If the user installs Xcode later, `open Package.swift` opens the SPM package as if it were an Xcode project, with full GUI debugger / previews.

**Tests note:** `swift test` does not work in this environment because Command Line Tools alone doesn't ship XCTest or `swift-testing`. Instead, the `TimapVerify` executable target runs assertion-style checks against `TimapCore`. **When you add new logic to `TimapCore`, add a check in `Sources/TimapVerify/main.swift`** — that's how regression coverage is maintained here. If the user later installs full Xcode, a parallel `TimapTests` XCTest target can be added, but until then `TimapVerify` is the source of truth.

## Architecture

### Three SPM targets

- `TimapCore` (library) — pure logic and data: `Teammate`, `AppState`, `TimeMath`, `TeamStore`, `Geo` (continents + projection + dotGrid), `LabelPlacer`, `CityCatalog`, formatting helpers. **No UI.** All public symbols are explicitly `public` so the executable targets can use them.
- `Timap` (executable) — SwiftUI app: `TimapApp` (the `MenuBarExtra` scene), all `Views/*`. Depends on `TimapCore`. Holds the only entry point (`@main`).
- `TimapVerify` (executable) — assertion-based checks for `TimapCore`. Run via `make verify`. Acts as the regression test suite until full Xcode is available.

The split is load-bearing: putting pure logic in `TimapCore` lets `TimapVerify` exercise it without booting SwiftUI.

### Single source of truth: `AppState`

`@MainActor final class AppState: ObservableObject` (in `TimapCore/Models/AppState.swift`) is the only stateful object the views read from. Three `@Published` properties (`team`, `hiddenIDs`, `hasOnboarded`) auto-persist via `didSet → TeamStore.save(...)`. A 1-minute `Timer.publish` in `TimapApp.body` calls `state.tick()` to advance `hostHour` in live mode (i.e., when `isLive == true`).

User dragging the slider/clicking a row strip calls `state.setHostHour(_:)` which sets `isLive = false`. The "Now" button in `HeaderView` calls `state.goLive()` which snaps back to current time and re-enables auto-tick.

`hostOffsetHours` is computed from the **You** teammate's IANA `tzIdentifier` via `TimeZone(identifier:).secondsFromGMT(for: Date())` — DST is handled automatically. Don't reintroduce hard-coded UTC offsets; `TimeZone` is the source of truth.

### Onboarding gate

First launch has no `isMe` teammate. `ContentView` checks `state.hasOnboarded` and renders `OnboardingView` until the user picks their city via `CityPickerView`. After that, `state.completeOnboarding(me:)` inserts the You teammate, sets `hasOnboarded = true`, and persistence is automatic.

### World map rendering

`WorldMapView` is a hybrid: `Canvas` for the dense parts (background, day glow, ~5000 land dots, terminator, sun) and SwiftUI views overlaid for pins and labels. The land dots are **precomputed once** as `ContinentData.landDotsViewBox` (a `static let` whose initializer runs ~5000 point-in-poly tests on first access, then caches) — don't compute them per draw or you'll regret it on slider drag.

Labels are positioned via `LabelPlacer.place(team:)` which sorts by latitude and pushes overlapping labels down by a minimum gap. Each `FloatingLabel` measures its own width via a `PreferenceKey` and uses `.position(centerX, centerY)` to anchor either the leading or trailing edge to the pin (decided by `placed.onRight`).

### Continent polygons

`Sources/TimapCore/Models/Geo.swift` holds ~200 anchor points across 13 polygons (Americas, Eurasia, Africa, Australia, Greenland, UK, Ireland, Japan, NZ, Indonesia, Madagascar, New Guinea, Philippines), traced clockwise. The web prototype iterated extensively on these — Beijing wasn't on land for a while, the UK polygon cut off Kent, Russia's arctic coast was a flat line. **If a city pin lands in the ocean**, the fix is to extend the relevant polygon, not move the city. Add a corresponding check in `TimapVerify` so the regression doesn't return.

The projection is equirectangular but cropped to lat 78°N…-56°S (skipping Antarctica) — see `Projection.latTop` / `latBot`. Aspect ratio 2.7:1. Sun longitude derives from UTC hour (`Projection.sunLongitude`).

### `MenuBarExtra` and dock policy

`TimapApp.init()` calls `NSApplication.shared.setActivationPolicy(.accessory)` to hide the Dock icon at runtime. `Info.plist` also sets `LSUIElement=YES` (written by the Makefile when bundling). Both are required: the runtime call covers `swift run` development, the Info.plist covers the proper `dist/Timap.app` bundle.

**Do not use `NSApp` (the global) inside `init()` — it's nil that early.** Use `NSApplication.shared` which creates the singleton on first access.

### Bundling pipeline

`make bundle` produces `dist/Timap.app` by:
1. `swift build -c release` → produces `.build/release/Timap` (binary) and `.build/release/Timap_TimapCore.bundle` (SPM resource bundle for `cities.json`).
2. Wraps both into `Timap.app/Contents/MacOS/` alongside a hand-written `Info.plist` (`LSUIElement=YES`, `LSMinimumSystemVersion=13.0`).

The resource bundle MUST sit next to the executable, since `Bundle.module` looks at the executable's directory. Don't move it to `Contents/Resources/` — `Bundle.module` won't find it there.

## Working with the prototype

When fixing visual bugs in the Swift app, **read the prototype's matching JSX file** to see the original spec. Map of correspondences:

- `prototype/shared.jsx` → `TimapCore/Models/TimeMath.swift` + `Utilities/Date+Format.swift`
- `prototype/bd-map-styles.jsx` (the `dotGrid` branch + `COAST` data) → `TimapCore/Models/Geo.swift`
- `prototype/direction-bd.jsx` → split across `Views/HeaderView`, `WorldMapView`, `TimeSliderView`, `TeamRowView`, `TeamRowsView`, `BestSlotsView`

Color palette is intentionally identical between prototype and Swift: `#ff8a5b` accent (orange), `#7ed957` green, `#f5b53d` amber, `#0d1420` background. Hex strings are converted via `Color(hex:)` in `TimapCore/Utilities/Color+Hex.swift`.

## What was deliberately scoped out of MVP

The user explicitly excluded these — don't reintroduce without asking:

- `Copy invite` button (entire feature dropped)
- `Core / Eng / PM` segmented control (no group management in v1)
- Mac App Store distribution / code signing
- Calendar / EventKit integration
- Multi-day view (only "today" matters for now)

## City data

`Sources/TimapCore/Resources/cities.json` ships ~80 major cities. To add more, append `{ name, country, flag, lat, lng, tz }` entries. The IANA `tz` identifier matters (it's what drives DST); look one up at https://www.iana.org/time-zones if uncertain. **Never invent a tz identifier — `TimeZone(identifier:)` returns nil on unknown strings and the app silently falls back to `.current`.**
